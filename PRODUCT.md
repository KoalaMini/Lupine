# Lupine 产品定义

> **From Solo to Symphony —— 从独奏到交响**

## 产品定位

Lupine 是一个多 Agent 协作开发框架。核心是一个"产品经理"角色的主Agent（Lupine），它不自己干所有活——它带领四个专业子Agent（探索器、规划器、执行器、评估器）协同完成开发任务。

## 核心理念

- **一个入口**：你只需跟 Lupine 对话，不需要在不同角色间手动切换
- **思考伙伴**：Lupine 不只是被动接收需求，它会挑战假设、带来新角度、带你发散再收敛，帮你找到"现阶段最适合你的方案"
- **专业分工**：四个子Agent各司其职，上下文隔离，工具限域
- **质量闭环**：评估器作为门禁，不合格的打回重做

## 架构

```
你 ──→ Lupine（狼王 / 产品经理）
          ├── 探索器  — 代码库浏览、Web搜索、文档阅读（只读）
          ├── 规划器  — 技术方案设计、架构决策
          ├── 执行器  — 写代码、写测试
          └── 评估器  — plan审查、code审查、门禁控制
```

## 角色职责

| 角色 | 类型 | 职责 |
|------|------|------|
| Lupine | 主Agent | 跟用户对话探讨、协调子Agent、产出PRODUCT.md + specs/ |
| 探索器 Explorer | 子Agent | 代码库探索、Web搜索竞品调研、文档阅读（只读） |
| 规划器 Planner | 子Agent | 基于确认的spec做技术方案设计，产出plans/ |
| 执行器 Executor | 子Agent | 拆task、写代码、写测试，产出tasks/ |
| 评估器 Evaluator | 子Agent | plan审查 + code审查，产出reviews/ |

## 目标用户

使用 AI Coding 工具（Claude Code、opencode 等）的开发者和技术负责人，希望从"人给指令→AI 写代码"的直给模式，升级为有角色分工、质量门禁、可追溯的团队协作模式。

## 成功标准

- 从需求到可合并 PR 的平均周期缩短 ≥ 30%
- 评估器门禁首次通过率 ≥ 80%
- 关键产出物（specs / plans / reviews）文档覆盖率达 100%
- 需求变更导致的返工显著减少
- 技术方案与最终代码实现的一致性可验证

## 范围边界

### 列入范围
- Lupine 主Agent 的定义与工作流规范
- 四子Agent（探索器/规划器/执行器/评估器）的定义与工作流
- 各阶段产出物模板、命名规范与目录结构
- 评估门禁 checklist 和审查流程
- 初始化脚本与项目骨架生成
- 与 Claude Code、opencode 等工具的集成

### 不做
- 不替代 Git / CI/CD 基础设施
- 不做 AI 模型训练或微调
- 不做 UI 界面（当前为 CLI + Markdown 文档驱动）
- 不做复杂 CLI 工具集（脚手架仅保留初始化职责）
- 不替代代码编辑器的 AI 补全功能
