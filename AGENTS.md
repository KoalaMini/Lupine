# Lupine Agent 集群配置

## 角色定义

| 角色 | 职责 | 方式 | 产出物 | 需用户参与 |
|------|------|------|--------|-----------|
| 分析器 Analyzer | 与用户对话、澄清需求、写需规 | 迭代式 | SPECS/{feature}.md | ✅ 每轮确认 |
| 规划器 Planner | 基于确认的需规做技术设计 | 一次性 | PLANS/{feature}.md | ❌ |
| 评估器 Evaluator | 在两个门禁节点做审查 | 一次性 | REVIEWS/{feature}-*.md | ❌ |
| 执行器 Executor | 拆 task、写代码、写测试 | 拆并行 | 代码 + 测试 | ❌ |

## 工作流规则

1. 分析器和规划器不可同时进行——需求必须先确认才能做设计
2. 评估器②未通过 → 打回执行器，附审查报告作为上下文
3. 评估器①未通过 → 打回规划器，附审查报告作为上下文
4. 评估器①发现需规遗漏 → 打回分析器，而非让规划器补
5. **不允许跳过评估阶段直接合并**
6. **PR 必须经人工审核后方可合并**（AI 评估器审查结果供参考，不替代人工审核）
7. 禁止 self-merge

## 例外处理

| 场景 | 处理方式 |
|------|---------|
| 需求变更 | 回到分析器，更新 SPECS，标记新版本号 |
| 实现中发现 plan 不可行 | 回到规划器修 plan，再经评估器①审查 |
| 紧急修复（如线上 bug） | 跳过调研+规划，直接执行+评估②，事后补 SPECS |
| 微小变更（如改文案） | 跳过调研和规划，直接执行后评估② |
