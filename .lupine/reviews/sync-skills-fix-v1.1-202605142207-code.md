## 问题清单

### 🔴 阻断性（1 个）

**B1. sync-skills.sh 注释与实际逻辑不一致**

- 位置：`scripts/sync-skills.sh:6`
- 当前注释：`# 仅当源码版本 > 已安装版本时执行安装。`
- 实际逻辑：版本号比较 + 内容哈希兜底
- 影响：误导后续维护者，与多源一致性检查冲突
- 修复：更新注释为 `# 当源码版本 > 已安装版本，或内容哈希不同时执行安装。`

### 🟡 建议性（2 个）

**S1. 测试脚本场景5描述与实际行为矛盾**

- 位置：`scripts/test-sync-skills.sh:135`
- 描述：`"场景5: 已安装版本更高（不应同步）"`
- 实际行为：`should_sync` 返回 `"内容变更 (2.0)"`（return 0，即**会同步**）
- 说明：当前逻辑以源文件为权威来源，版本号降级场景也会因哈希不同触发同步。这是设计意图（源文件为准），但测试描述"不应同步"与期望输出 `"内容变更 (2.0)"` 自相矛盾。
- 建议：要么修改描述为 `"场景5: 已安装版本更高但内容不同（以源为准，触发同步）"`，要么在 `should_sync` 中增加版本号降级保护逻辑。

**S2. `read_skill_version` 对带引号版本号处理不够健壮**

- 位置：`scripts/sync-skills.sh:18-22`
- 问题：`grep '^version:' | sed 's/version:[[:space:]]*//'` 会保留 YAML 值两侧的引号（如 `"1.1"`），`ver_gt` 函数在算术比较时可能行为异常
- 当前 skill 文件无引号，暂无影响，但属于潜在隐患
- 建议：`sed` 后加 `| tr -d '"'` 去除引号
