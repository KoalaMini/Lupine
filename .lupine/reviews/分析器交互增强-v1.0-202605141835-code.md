## 七、WARNINGs

### WARNING #1：SKILL.md 长度显著增加

| 指标 | 旧版 | 新版 | 变化 |
|------|------|------|------|
| 行数 | ~69 行 | 266 行 | +286% |
| 章节数 | 4 个 | 7 个 + Visual Companion | +3 个 |

**影响**：SKILL.md 作为 AI session 的上下文入口，长度增加可能导致：
- 上下文窗口占用增加
- AI 对关键信息的注意力分散
- 多轮对话后可用上下文减少

**缓解**：当前版本中 Visual Companion 章节占比较大（~70 行），建议后续迭代中将 SVG 示例模板和详细 CSS 规范抽取到独立参考文件（如 `.claude/skills/lupine-analyzer/SKETCH-REFERENCE.md`），SKILL.md 中仅保留概要引用。

**严重级别**：低。不影响功能正确性，仅影响未来可维护性。

### WARNING #2：分块确认设计和 YAGNI 自检未作为独立章节

Plan 中规划的结构包含独立的 `## 分块确认设计` 和 `## YAGNI 与自检` 章节，但实际 SKILL.md 中：
- 分块确认仅在阶段四中描述
- YAGNI 仅在阶段五中描述

**影响**：这两个能力被嵌套在工作流中，用户（和 AI）可能难以快速定位到相关规范。

**缓解**：当前内容在阶段四/五中已足够详细，不影响功能。建议后续版本补充独立章节，提升可检索性。

**严重级别**：低。不影响功能正确性。
