# 项目结构重组设计

## 版本
- v1.0
- 日期: 2026-05-13

---

## 背景与问题

当前项目根目录积累了多个职责不同的文件，导致入口不够聚焦：

1. **CLAUDE.md 职责过重** — 同时承担"AI 入口"和"项目宪法"双重角色，包含技术栈、Git 规范、编码约定等大量内容
2. **规范分散** — Git 规范、技术栈、编码规范、Agent 角色定义、评估标准等散落在根目录不同文件中
3. **命名不一致** — `specs/` 为小写，`PLANS/`、`REVIEWS/`、`TASKS/` 为大写
4. **缺乏框架自管理空间** — Lupine 作为一套开发流程框架，自身的需求迭代没有独立的管理空间

## 目标

1. 让 CLAUDE.md 回归轻量索引，成为 AI session 的第一入口
2. 聚合所有规范类文件到统一位置
3. 统一全项目文件/文件夹命名规范
4. 建立 `.lupine/` 空间，用 Lupine 流程管理框架自身的迭代

## 设计方案

### 1. 根目录：三层入口

根目录只保留真正的"入口文件"，每个文件有明确的读者和职责：

| 文件 | 读者 | 职责 |
|------|------|------|
| `CLAUDE.md` | AI | 索引：告诉 AI 读哪些文件、按什么顺序、扮演什么角色 |
| `README.md` | 人 | 项目介绍、快速开始、设计理念 |
| `VISION.md` | AI + 人 | 产品总纲：目标用户、价值主张、成功标准、范围边界 |

### 2. rules/：规范仓库

所有"你应该怎么做"的文件聚合到 `rules/`：

| 文件 | 来源 | 内容 |
|------|------|------|
| `rules/git.md` | 原 CLAUDE.md 提取 | 分支策略、Commit 规范、PR 流程 |
| `rules/coding.md` | 原 CLAUDE.md 提取 | 技术栈、编码规范、目录约定 |
| `rules/agents.md` | 原 `AGENTS.md` 移动 | 四角色定义 + 工作流规则 |
| `rules/evals.md` | 原 `EVALS.md` 移动 | 评估门禁 checklist |

### 3. .lupine/：框架自管理

Lupine 框架自身的需求、设计、审查、任务放入 `.lupine/`，与面向用户的成品解耦：

```
.lupine/
├── specs/       # 框架级需求规格
├── plans/       # 框架级技术设计
├── reviews/     # 框架级审查报告
└── tasks/       # 框架级任务跟踪
```

内部结构和命名规范与外部成品完全一致。

### 4. 命名规范统一

全部文件夹和文件统一使用**小写 + 连字符**：

- `specs/` ✅（已符合）
- `plans/` ← 原 `PLANS/`
- `reviews/` ← 原 `REVIEWS/`
- `tasks/` ← 原 `TASKS/`

文件命名沿用现有约定：`{功能名称}-v{版本号}-{YYYYMMDD}.md`

## 新的完整文件结构

```
project-root/
├── CLAUDE.md              # AI 索引入口
├── README.md              # 面向人的项目介绍
├── VISION.md              # 产品总纲
├── rules/
│   ├── git.md             # Git 协作规范
│   ├── coding.md          # 技术栈与编码规范
│   ├── agents.md          # Agent 角色定义与工作流
│   └── evals.md           # 评估门禁标准
├── specs/                 # 功能级需求规格（成品）
│   └── {功能名称}-v{版本号}-{YYYYMMDD}.md
├── plans/                 # 技术设计方案（成品）
│   └── {YYYYMMDD}-{功能名称}.md
├── reviews/               # 审查报告（成品）
│   └── {功能名称}-v{版本号}-{YYYYMMDD}-{类型}.md
├── tasks/                 # 任务跟踪（成品）
│   └── {YYYYMMDD}-{功能名称}.md
├── .lupine/               # 框架自身开发管理
│   ├── specs/
│   ├── plans/
│   ├── reviews/
│   └── tasks/
└── bin/
    └── lupine-init        # 初始化脚本
```

## CLAUDE.md 的新内容

CLAUDE.md 从"项目宪法"转变为"AI 导航索引"，内容应包含：

1. **项目简介** — 一句话说明项目是什么
2. **AI 阅读顺序** — 告诉 AI 应该按什么顺序读取哪些文件
3. **当前角色指引** — 根据用户提示判断当前应该扮演什么角色（分析器/规划器/评估器/执行器）
4. **快速导航** — 各文件的绝对路径和用途

具体内容在实现阶段由执行器编写。
