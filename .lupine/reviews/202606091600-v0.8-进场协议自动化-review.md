## 2. 发现问题

### 🔴 #1 [CRITICAL] update.js 未定义/导入 askQuestion

**文件**：`packages/lupine/src/update.js`  
**位置**：第 277 行、第 294 行  
**问题**：两处调用 `askQuestion(...)`，但该函数仅在 `init.js` 中定义（第 16 行），`update.js` 中既未定义也未导入。运行时当 ARCHITECTURE.md 发生变更（或有新内容）时会抛出 `ReferenceError: askQuestion is not defined`。

**影响**：`lupine update` 命令在涉及 ARCHITECTURE.md 变化时必定崩溃。

**建议**：将 `askQuestion` 提取到公共模块（如 `utils.js`），或在 `update.js` 中也定义此函数。推荐前者以避免重复代码。

### 🟠 #2 [MAJOR] ARCHITECTURE.md 输出格式与 REQ-4 不符

**文件**：`packages/lupine/src/probe.js` — `generateArchitectureMdForAll()`  
**位置**：第 353–364 行  
**问题**：Spec REQ-4 明确定义的 ARCHITECTURE.md 结构包含四段：

```markdown
# 项目架构

## 产品架构
[多 repo 合并描述]

## 技术架构
[多 repo 分组列出，每个 repo 独立段落]

## 目录结构
[多 repo 分别展示]

## 基础设施
[多 repo 命令合并列表]
```

但当前实现仅将各 repo 的 `generateArchitectureMd()` 输出用 `---` 连接，缺少以下内容：
- 顶层 `# 项目架构` 标题
- `## 产品架构` 多 repo 合并描述段
- `## 技术架构` 分组结构（直接平铺 repo）
- `## 基础设施` 命令合并列表
- 各 section 之间的层级嵌套

**影响**：生成的文件与 spec 定义不一致，用户期望的结构化信息缺失。

**建议**：重构 `generateArchitectureMdForAll` 使其按 spec 的四段式输出。可在 `probeRepositories` 返回后做一次整体编排：
1. 从所有 result 中提取产品级描述（目前 probe 未采集这部分信息，需扩展）
2. 将各 repo 纳入 `## 技术架构` 子段落
3. 将各 repo 的 dirTree 归入 `## 目录结构`
4. 汇总各 repo 的 scripts 为 `## 基础设施`

### 🟡 #3 [MINOR] 预览未实现 diff 风格

**文件**：`packages/lupine/src/init.js`（第 174–176 行）、`update.js`（第 269–274 行）  
**问题**：Spec 要求"终端 diff 风格（新增行绿色、删除行红色、无变化不展示）"，但当前实现仅输出行数变化和内容截取预览，未做行级差异高亮。

**影响**：用户无法直观看到具体哪些行有变化，降低了确认效率。

**建议**：可引入 `diff` 库（如 `diff` 或 `jsdiff`）对新旧内容做逐行 diff，用 ANSI 颜色码输出差异块。轻量方案：展示变化的前 5 行和后 5 行上下文。
