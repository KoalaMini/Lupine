# lupine-init 重构：模块抽离 — 技术方案

> 基于 spec: `specs/lupine-init重构-v1.0-20260514.md`

---

## 1. 架构与组件划分

### 文件结构与职责

```
bin/lupine-init              (编排层 — 精简)
  ├─ source scripts/init-check.sh       (防御检查)
  │    └─ init_check()
  ├─ source scripts/init-git-flow.sh    (Git 交互)
  │    └─ init_git_flow()
  ├─ source scripts/init-exec.sh        (执行落地)
  │    └─ init_exec()
  └─ 内联：banner、路径解析、收尾打印
```

### 通信契约

所有模块通过 **环境变量 + 返回值** 通信，不写全局变量、不传复杂参数：

| 环境变量 | 设置者 | 消费者 | 说明 |
|---------|--------|--------|------|
| `LUPINE_FRAMEWORK_DIR` | 顶层 `bin/lupine-init` | `init_check`, `init_exec` | 框架根目录 |
| `INIT_DIR` | 顶层 `bin/lupine-init` | `init_git_flow`, `init_exec` | 目标项目目录 |
| `ASSETS_DIR` | 顶层 `bin/lupine-init` | `init_check` | 模板目录路径 |

### 函数签名

```bash
init_check()    # return 0=通过, 1=失败(自动 exit)
init_git_flow() # return 0=已确认, 1=用户跳过
init_exec()     # return 0=成功, 1=失败
```

## 2. 各模块详细设计

### 2.1 `scripts/init-check.sh` — 防御性检查

```bash
init_check() {
    if [ ! -d "$ASSETS_DIR" ]; then
        echo "Error: 模板目录不存在: $ASSETS_DIR" >&2
        echo "请检查 Lupine 框架是否正确安装。" >&2
        return 1
    fi
    return 0
}
```

职责单一：仅检查模板目录存在性。

若未来需要增加检查项（如依赖命令存在性），直接在该函数内追加。

### 2.2 `scripts/init-git-flow.sh` — Git Flow 交互

将原 `bin/lupine-init` 第 34-49 行的 `read -p` + `case` 逻辑完整搬入：

```bash
init_git_flow() {
    echo ""
    echo "🐺 Git 分支策略"
    read -p "  是否采用 Lupine 推荐的 Git Flow 分支策略（master/dev/release/feat/fix）？(Y/n) " choice
    case "$choice" in
        [Nn]*)
            cat > "$INIT_DIR/rules/git.md" <<'GITEOF'
# Git 协作规范

（此项目未采用 Lupine 推荐的 Git Flow，请根据团队规范自行补充）
GITEOF
            echo "  已跳过 Git Flow 配置，rules/git.md 已置空"
            ;;
        *)
            echo "  已应用 Git Flow 分支策略"
            ;;
    esac
    return 0
}
```

不修改 `git.md` 模板内容，仅搬运。

### 2.3 `scripts/init-exec.sh` — 执行落地

```bash
init_exec() {
    mkdir -p "$INIT_DIR"
    cp -r "$ASSETS_DIR/"* "$INIT_DIR/"

    if [ -f "$LUPINE_FRAMEWORK_DIR/scripts/install-skills.sh" ]; then
        echo "🐺 正在安装 Lupine Skills..."
        "$LUPINE_FRAMEWORK_DIR/scripts/install-skills.sh"
    fi
    return 0
}
```

### 2.4 `bin/lupine-init` — 编排入口（重构后）

```bash
#!/bin/bash
set -e

PROJECT_NAME="${1:-my-project}"
INIT_DIR="${2:-.}"

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
LUPINE_FRAMEWORK_DIR="$(cd "$SCRIPT_DIR/.." && pwd)"
ASSETS_DIR="$LUPINE_FRAMEWORK_DIR/assets"

# 加载模块
. "$LUPINE_FRAMEWORK_DIR/scripts/init-check.sh"
. "$LUPINE_FRAMEWORK_DIR/scripts/init-git-flow.sh"
. "$LUPINE_FRAMEWORK_DIR/scripts/init-exec.sh"

# Banner
echo "🐺 Lupine 项目初始化"
echo "   项目名: $PROJECT_NAME"
echo "   目录: $INIT_DIR"
echo ""

# 1. 防御检查（显式检查返回值，避免 set -e 对函数 return 行为歧义）
init_check || exit 1

# 2. 创建目录 + 复制模板
init_exec

# 3. Git Flow 交互
init_git_flow

# 4. 收尾
echo "✅ 初始化完成"
echo ""
echo "文件结构:"
find "$INIT_DIR" -maxdepth 2 -type f | sort
echo ""
echo "下一步:"
echo "  1. 编辑 rules/coding.md，填入项目技术栈和约定"
echo "  2. 编辑 README.md，补充项目介绍"
echo "  3. 启动分析器，产出 VISION.md 和 specs"
echo "  4. 使用快捷指令切换角色: /lupine-analyzer, /lupine-planner, /lupine-evaluator, /lupine-executor"
```

## 3. 数据流

```
用户输入参数
     │
     ▼
参数解析 ─→ 设置 LUPINE_FRAMEWORK_DIR / INIT_DIR / ASSETS_DIR
     │
     ├─ init_check     → 检查 ASSETS_DIR 是否存在 → 失败则 exit
     │
     ├─ init_exec      → mkdir + cp (读 LUPINE_FRAMEWORK_DIR, INIT_DIR, ASSETS_DIR)
     │                    └─ 可选: 调用 scripts/install-skills.sh
     │
     ├─ init_git_flow  → read 用户输入 → 写 rules/git.md
     │
     └─ 收尾打印       → find 文件树 + 输出 next steps
```

## 4. 错误处理策略

| 错误场景 | 处理方式 | 责任模块 |
|---------|---------|---------|
| ASSETS_DIR 不存在 | `init_check` return 1 → 主脚本 `init_check || exit 1` 兜底退出 | init-check |
| source 文件不存在 | bash 自动报错 + 终止（`set -e`） | 编排层 |
| mkdir / cp 失败 | `set -e` 自动终止 | init-exec |
| install-skills.sh 不存在 | `-f` 判断跳过，不报错 | init-exec |
| 用户 Ctrl+C | 自然终止，无特殊处理 | — |

注意：bash 的 `set -e` 不会自动终止函数 `return 1` 的场景，主脚本需对 `init_check` 做返回值检查：

```bash
init_check || exit 1
```

## 5. 迁移策略

直接原地替换：

1. 创建 `scripts/init-check.sh`、`scripts/init-git-flow.sh`、`scripts/init-exec.sh`
2. 用新的编排版本替换 `bin/lupine-init`
3. 冒烟测试：在临时目录运行 `bin/lupine-init test-project /tmp/lupine-test`，验证：
   - 目录结构正确
   - `rules/git.md` 根据 Y/n 正确生成
   - skills 安装正常
   - 文件树展示正常
4. 无风险后删除临时目录
