### Phase 4：lupine-init 重构

#### Task 4.1：移动并重写 lupine-init

```bash
# 先移动
mv bin/lupine-init framework/init/lupine-init.sh
```

重写 `framework/init/lupine-init.sh` 使其：

1. 接受 `--mode peer|embedded` 和 `--platform opencode|claude|auto` 参数
2. 读取 `template.yaml`（使用 Python 解析 YAML，更可靠）
3. 遍历 entries 生成目标文件
4. 支持 `mode: additive`（追加到已有 CLAUDE.md）
5. 支持 `generate: agent`（拼装 frontmatter + prompt）
6. 输出文件清单

**关键实现细节：**

**additive 模式：**
```bash
# 伪代码
if mode == "additive" && target file exists; then
    # 先备份
    cp "$target" "$target.bak"
    # 在末尾追加 Lupine 块（有分隔线）
    echo "" >> "$target"
    echo "---" >> "$target"
    echo "> 本项目使用 Lupine 多 Agent 流水线..." >> "$target"
    cat "$rendered_content" >> "$target"
fi
```

**agent 渲染：**
```bash
# 伪代码
prompt=$(cat "templates/agents/${agent_name}.prompt")
frontmatter=$(construct from template.yaml agents.${agent_name}.${platform})
echo "---" > "$target"
echo "$frontmatter" >> "$target"
echo "---" >> "$target"
echo "$prompt" >> "$target"
```

**幂等性：** 检测目标文件是否已包含 Lupine 块，避免重复追加。
