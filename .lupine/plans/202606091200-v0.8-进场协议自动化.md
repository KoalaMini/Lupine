## 测试用例

### 边界值测试用例

| 编号 | 测试对象 | 场景 | 输入 | 预期结果 |
|------|---------|------|------|---------|
| B-1 | `probeRepository` | 空路径 | `""` | 抛出明确错误或返回降级结果 |
| B-2 | `probeRepository` | 不存在路径 | `/nonexistent/path` | 返回 lang="未识别"，不抛异常 |
| B-3 | `probeRepository` | 文件而非目录 | 指向一个 `.txt` 文件 | 返回 lang="未识别"，不抛异常 |
| B-4 | `probeRepositories` | 空数组 | `[]` | 返回 `[]` |
| B-5 | `probeRepositories` | 混合有效/无效路径 | `['/valid/repo', '/invalid']` | 有效路径正常探查，无效路径降级 |
| B-6 | `probeRepository` | 无权限目录 | 一个无 r/x 权限的目录 | 返回 lang="未识别"，不抛异常 |
| B-7 | `generateArchitectureMd` | 空结果 | lang="未识别" 的 ProbeResult | 生成包含"技术栈: 未识别"的 Markdown |
| B-8 | init — 无仓库 | 用户不输入任何 repo | repos=[] | 跳过探查，不生成 ARCHITECTURE.md |
| B-9 | init — 用户拒绝写入 | 确认环节输入 n | askQuestion 返回 n | 不生成 ARCHITECTURE.md，流程继续 |
| B-10 | update — ARCHITECTURE.md 不存在 | 首次 update | 读取 oldContent 失败 | 展示预览，确认后创建 |
| B-11 | update — 无变化 | 仓库未变动 | 新旧内容一致 | 跳过，输出 "无变化" |
