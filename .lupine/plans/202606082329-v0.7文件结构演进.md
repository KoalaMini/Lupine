### T4: rules/coding.md 增强

- [x] **Step 4.1：在 `.lupine/rules/coding.md` 末尾增加"Commit 质量指引"一节**
  - Target: `.lupine/rules/coding.md`
  - 新增一节 `## Commit 质量指引`，内容：
    - **核心原则：可回退**——每次完成一个可验证的任务节点后 commit
    - commit message 格式：`<动词>: <做了什么>（原因：<为什么>）`
    - 好示例：
      - `feat: 新增 lupine init 命令（原因：用户需要一键初始化项目工作区）`
      - `fix: 修复 init 模板占位符替换 bug（原因：{project_name} 未正确渲染）`
      - `refactor: 抽取 checksum 模块（原因：generate.js 体积超 500 行）`
    - 不好示例：
      - `fix bug`（没说什么 bug）
      - `update`（没说什么、为什么）
      - `wip`（不可回退）
    - 回退检查清单：commit 前自查——"如果我 3 小时后回退到这个 commit，能知道当时在干什么吗？"
  - 验收：`rules/coding.md` 末尾包含 `## Commit 质量指引` 节，有好/不好示例
