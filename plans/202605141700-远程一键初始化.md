## 1. 总体架构

```

    │
    └── tag: v* ──→ GitHub Actions
```

| 文件 | 类型 | 说明 |

| `install.sh` | 新增 | 远程安装脚本，放在仓库根目录，用户 curl 下载后执行 |

| `bin/lupine-init` | 修改 | 支持从 Release tarball 解压后的环境运行 |

## 2. 关键设计决策

### 2.1 install.sh 的自包含设计

`install.sh` 必须是**完全自包含**的，不能依赖仓库中的其他文件（因为用户只下载了这个脚本）。所有逻辑（版本解析、下载、解压、替换）都在单个文件中实现。

```
install.sh
├── 环境检查（curl、写权限）
├── 版本解析（GitHub API → fallback 到固定 URL）
├── tarball 下载（到 /tmp）
├── 冲突检测（检查已有 Lupine 文件）
├── 解压与嵌入（tar -xz + cp）
├── 占位符替换（sed）
├── 清理临时文件
└── 输出使用指引
```

### 2.2 构建脚本的防御性设计

`build-release.sh` 采用**显式白名单**复制，而非黑名单排除。即使仓库新增目录，只要不在白名单内就不会混入产物。

```bash
# 只复制以下目录/文件
cp -r bin assets scripts skills README.md "$BUILD_DIR/"
# 其余所有内容（plans/ reviews/ specs/ .github/ 等）自动排除
```

### 2.3 版本解析降级策略

```
优先：GitHub API 查询 latest release
    ↓ 403/限流
降级：直接访问 https://github.com/{owner}/{repo}/releases/latest/download/lupine-context.tar.gz
    ↓ 仍失败
报错：提示用户手动指定 VERSION 或下载 tarball
```

## 3. 文件详细设计

### 3.1 `install.sh`

**位置**：仓库根目录（master 分支）
**语言**：POSIX sh（兼容 dash、bash、zsh）
**体积目标**：≤ 5KB

```bash
#!/bin/sh
set -e

# 配置
OWNER="koalamini"
REPO="lupine"
TARBALL_NAME="lupine-context"

# 环境变量
VERSION="${VERSION:-}"
FORCE="${FORCE:-}"
DRY_RUN="${DRY_RUN:-}"

# 1. 环境检查
# 2. 版本解析（API → fallback）
# 3. 下载 tarball 到 /tmp
# 4. 冲突检测
# 5. 解压嵌入
# 6. 占位符替换
# 7. 清理 + 输出指引
```

**关键函数**：
- `check_env()`：检查 curl、写权限
- `resolve_version()`：调用 GitHub API，限流时降级
- `download_tarball()`：curl 下载，重试 3 次
- `check_conflicts()`：检查 rules/agents.md 等
- `extract_and_install()`：解压并复制到当前目录
- `replace_placeholders()`：sed 替换 {项目名称}
- `print_guide()`：输出使用指引

### 3.2 `scripts/build-release.sh`

**位置**：`scripts/` 目录
**调用方式**：`./scripts/build-release.sh [VERSION]`
**CI 调用**：`./scripts/build-release.sh "${GITHUB_REF#refs/tags/}"`

```bash
#!/bin/bash
set -e

VERSION="${1:-$(git describe --tags --always)}"
BUILD_DIR=$(mktemp -d)

echo "🏗️  构建 Lupine Release v$VERSION"

# 显式白名单复制
mkdir -p "$BUILD_DIR/lupine-context-$VERSION"
cp -r bin assets scripts skills README.md "$BUILD_DIR/lupine-context-$VERSION/"

# 清理 assets 内可能混入的项目级文件
rm -f "$BUILD_DIR/lupine-context-$VERSION/assets/README.md" 2>/dev/null || true

# 版本标记
echo "$VERSION" > "$BUILD_DIR/lupine-context-$VERSION/.lupine-version"

# 打包
tar -czf "lupine-context-$VERSION.tar.gz" -C "$BUILD_DIR" "lupine-context-$VERSION"

# 清理
rm -rf "$BUILD_DIR"

echo "✅ 构建完成: lupine-context-$VERSION.tar.gz"
```

### 3.3 `.github/workflows/release.yml`

**触发**：`push` 事件 + `tags: ['v*']`
**权限**：需要 `contents: write` 来创建 Release

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  build-and-release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Build Release
        run: |
          VERSION=${GITHUB_REF#refs/tags/}
          ./scripts/build-release.sh "$VERSION"

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v1
        with:
          files: lupine-context-*.tar.gz
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### 3.4 `bin/lupine-init` 兼容性修改

当前 `bin/lupine-init` 假设从本地仓库运行（通过 `$SCRIPT_DIR` 定位框架目录）。需要调整为**同时支持两种模式**：

1. **本地模式**（已有）：从克隆的仓库运行
2. **Release 模式**（新增）：从 tarball 解压后的目录运行

检测逻辑：
```bash
if [ -f "$SCRIPT_DIR/../scripts/init-check.sh" ]; then
    # 本地模式：框架目录在上一级
    LUPINE_FRAMEWORK_DIR="$(cd "$SCRIPT_DIR/.." && pwd)"
else
    # Release 模式：当前目录就是框架目录
    LUPINE_FRAMEWORK_DIR="$SCRIPT_DIR/.."
fi
```

实际上更简单的方案：Release tarball 中的目录结构保持与本地仓库一致（`bin/`、`scripts/`、`assets/` 同级），这样 `lupine-init` 的路径逻辑无需修改。

## 5. 错误处理与边界情况

| 场景 | 处理策略 |
|------|---------|
| GitHub API 403（限流） | 降级到 `latest/download/lupine-context.tar.gz` 固定 URL |
| 下载超时 | curl `--retry 3 --retry-delay 2`，仍失败则 exit 1 |
| 当前目录不可写 | 前置检查，exit 1 |
| 已有 Lupine 文件（未 FORCE） | 列出冲突文件，exit 1，提示加 FORCE=1 |
| 占位符替换失败 | warning 提示用户手动编辑，不阻断 |
| tarball 校验 | 不额外做 SHA 校验（GitHub HTTPS 已提供传输层安全） |

## 7. 部署流程

```
1. 开发完成后，合并到 dev → PR → master
2. 在 master 打 tag: git tag -a v1.0.0 -m "release v1.0.0"
3. push tag: git push origin v1.0.0
4. GitHub Actions 自动触发：
   - 执行 build-release.sh
   - 创建 GitHub Release v1.0.0
   - 上传 lupine-context-v1.0.0.tar.gz
5. 用户即可使用：
   curl -fsSL https://raw.githubusercontent.com/koalamini/lupine/master/install.sh | bash
```
