# 需求规格说明书：远程一键初始化

> **功能名称**：远程一键初始化（curl-install）
> **版本**：v1.0
> **日期**：2026-05-14 16:00
> **状态**：待用户确认

---

## 1. 功能概述

提供类似 `curl -fsSL https://lupine.dev/install | bash` 的一键安装体验，让 Lupine 的上下文模块（`rules/`、`CLAUDE.md`、目录骨架等）能够**直接注入到用户现有的项目中**，无需用户先克隆 Lupine 仓库。

### 核心价值
- **零前置依赖**：用户不需要 `git clone` Lupine 仓库
- **一键嵌入**：一条命令把 AI 协作规范注入现有项目
- **版本可控**：默认跟随 GitHub Release 最新版，可指定版本

---

## 2. 用户旅程

```
用户在自己的项目目录中执行：
  curl -fsSL https://get.lupine.dev/install | bash

系统响应：
  1. 检测环境（curl、git、目录权限）
  2. 查询 GitHub Release API 获取最新版本
  3. 下载对应版本的 assets 压缩包
  4. 解压并复制到当前目录
  5. 替换模板占位符（项目名称等）
  6. 输出使用指引
```

---

## 3. 功能范围

### 3.1 明确列入范围

| 功能点 | 说明 |
|--------|------|
| 远程安装脚本 | 单个 `install.sh`，托管在可公开访问的 CDN（如 GitHub raw / jsDelivr） |
| 版本解析 | 默认获取 GitHub Release 最新版；支持 `VERSION=v1.2.0` 环境变量指定 |
| 资源下载 | 从 GitHub Release 下载 `lupine-context-v{VERSION}.tar.gz` |
| 嵌入安装 | 将 assets 解压到当前工作目录（非子目录），与现有文件合并 |
| 占位符替换 | 自动提取当前目录名作为 `{项目名称}`，替换 `CLAUDE.md` 和 `README.md` 中的占位符 |
| 冲突检测 | 若目标目录已存在 Lupine 文件（如 `rules/agents.md`），提示用户并询问是否覆盖 |
| 使用指引 | 安装完成后打印清晰的下一步操作说明 |
| 离线备用 | 若 GitHub API 限流，提供降级逻辑（直接尝试已知最新版 URL） |

### 3.2 明确不做（防止范围蔓延）

| 功能点 | 说明 |
|--------|------|
| 不管理 Lupine 版本更新 | 安装脚本只负责首次嵌入，后续更新需用户手动执行或另行设计 `lupine upgrade` |
| 不替代包管理器 | 不发布到 npm、Homebrew、apt 等，只提供 `curl \| bash` |
| 不做全局安装 | 不安装到 `~/.lupine/` 或系统 PATH，只嵌入当前项目 |
| 不做交互式配置 | Git Flow 分支策略等交互配置改为默认值+文档说明，不在 pipe 脚本中交互询问 |
| 不保留 specs/plans 模板 | 用户项目不需要需求文档模板，只保留 `specs/`、`plans/`、`reviews/`、`tasks/` 空目录（带 `.gitkeep`） |

## 4. 详细需求

### 4.1 安装脚本行为

```bash
# 默认行为：安装最新版到当前目录
curl -fsSL https://get.lupine.dev/install | bash

# 指定版本
curl -fsSL https://get.lupine.dev/install | VERSION=v1.2.0 bash

# 强制覆盖（非交互）
curl -fsSL https://get.lupine.dev/install | FORCE=1 bash

# 仅下载不安装（调试用）
curl -fsSL https://get.lupine.dev/install | DRY_RUN=1 bash
```

### 4.2 执行流程

```
1. 环境检查
   - 检查 curl 是否可用（不可用则提示并退出）
   - 检查当前目录是否有写权限
   - 检查当前目录是否在 git 仓库中（warning 提示，不阻断）

2. 版本解析
   - 若环境变量 VERSION 存在，使用指定版本
   - 否则调用 GitHub API: GET /repos/{owner}/{repo}/releases/latest
   - 提取 tag_name 作为版本号

3. 资源下载
   - 构造下载 URL: https://github.com/{owner}/{repo}/releases/download/{VERSION}/lupine-context-{VERSION}.tar.gz
   - 下载到临时目录（如 /tmp/lupine-install-XXXX）
   - 验证 HTTP 200，否则报错退出

4. 冲突检测
   - 检查当前目录是否已存在 `rules/agents.md` 或 `CLAUDE.md`
   - 若存在且 FORCE=1，直接覆盖
   - 若存在且 FORCE 未设置，打印冲突文件列表并退出（exit 1），提示用户加 FORCE=1 重试

5. 解压与嵌入
   - 解压 tar.gz 到临时目录
   - 复制文件到当前目录（保留目录结构）
   - 替换占位符：
     - `{项目名称}` → 当前目录名（basename $PWD）
     - `{一句话描述项目}` → 留空或提示用户手动编辑

6. 收尾与指引
   - 清理临时文件
   - 打印成功消息和文件清单
   - 打印下一步操作指引（见 4.3）
```

### 4.3 安装后指引输出示例

```

位置: /Users/alice/my-project/
```

### 4.4 错误处理

| 场景 | 行为 |
|------|------|
| curl 不可用 | 退出码 1，提示 "请先安装 curl" |
| GitHub API 限流（403） | 降级尝试下载最新已知版本的 assets，若失败则提示用户手动指定版本 |
| 下载 404 | 退出码 1，提示 "版本 {VERSION} 不存在，请检查" |
| 目录无写权限 | 退出码 1，提示当前目录不可写 |
| 文件冲突（未加 FORCE） | 退出码 1，列出冲突文件，提示加 FORCE=1 强制覆盖 |
| 网络超时 | 重试 3 次（间隔 2s），仍失败则退出 |

## 5. 安全与约束

### 5.1 安全设计

- **最小权限原则**：脚本只操作当前目录，不访问用户 home 以外的路径
- **显式确认**：默认不覆盖已有文件，需用户显式加 `FORCE=1`
- **可审计**：脚本执行前打印即将下载的 URL 和版本号，用户可见
- **临时文件**：所有下载内容先放到 `/tmp`，安装完成后清理

### 5.2 约束

- 脚本需兼容 POSIX sh（不依赖 bash 特有语法），确保在 Alpine 等精简系统可用
- GitHub API 调用需处理 rate limit（未认证 60 次/小时）
- Release assets 命名规范：`lupine-context-v{VERSION}.tar.gz`
