# 构建交付

> npm 脚手架 CLI、模板同步与版本管理、双渠道发布
> 版本：v1.0

---

## 1. 概述

构建交付模块负责 Lupine 框架的分发和安装。用户通过 npm 安装 CLI 工具，一键初始化项目工作区，并通过模板同步保持与上游框架的更新。

---

## 2. npm 脚手架 CLI

### 2.1 命令体系

```bash
# 初始化新项目
npx lupine init <project-name>

# 更新现有项目
npx lupine update

# 查看版本
npx lupine --version
```

### 2.2 init 命令

**功能**：
1. 创建 `.lupine/` 工作区目录
2. 从模板仓库复制基础文件（PRODUCT.md、rules/、specs/、CLAUDE.md 等）
3. 生成 `.lupineconfig.json`：
   ```json
   {
     "name": "<project-name>",
     "version": "0.1.0",
     "templateVersion": "x.y.z",
     "repoUrl": "..."
   }
   ```
4. 可选：安装推荐的 MCP/Skill 依赖

**交互流程**：
```
$ npx lupine init my-project
? 选择模板类型: [框架项目 / 应用项目 / 库项目]
? 是否安装推荐 MCP? [Y/n]
✓ 已创建 .lupine/ 目录
✓ 已复制模板文件
✓ 已生成 .lupineconfig.json
```

### 2.3 update 命令

**功能**：
1. 读取 `.lupineconfig.json` 获取当前模板版本
2. 对比远程最新版本
3. 显示变更差异（新增/修改/删除的文件列表）
4. 用户确认后合并变更
5. 更新 `.lupine-version`

**保护机制**：
- 用户修改过的文件不会自动覆盖，会提示冲突
- 支持 `--dry-run` 预览变更
- 支持 `--force` 强制覆盖（慎用）

## 3. 模板同步与版本管理

### 3.1 模板仓库结构

```
lupine-templates/
├── base/                    ← 基础模板（所有项目通用）
│   ├── PRODUCT.md.template
│   ├── rules/
│   └── specs/
├── framework/               ← 框架项目模板
├── app/                     ← 应用项目模板
└── lib/                     ← 库项目模板
```

### 3.2 同步策略

**文件级同步**：
- 模板文件以 `.template` 结尾，支持变量替换（如 `{{projectName}}`）
- 非模板文件直接复制
- 删除检测：远程删除的文件，本地也标记为待删除

**冲突处理**：
- 如果本地文件被用户修改过，update 时提示冲突
- 用户可选择：保留本地 / 使用远程 / 手动合并

### 3.3 版本管理

- **模板版本**：独立于框架版本，语义化版本号（如 `1.2.3`）
- **版本锁定**：`.lupineconfig.json` 中记录 `templateVersion`
- **版本检查**：update 时自动检查是否有新版本

## 4. 双渠道发布

### 4.1 npm Registry

**发布内容**：
- `lupine` CLI 包（包含 init/update 命令）
- 模板文件（内嵌在包中或作为依赖）

**版本策略**：
- 遵循语义化版本（SemVer）
- 每次发布更新 `package.json` 版本号
- 发布前运行测试套件

### 4.2 GitHub Release Tarball

**发布内容**：
- 完整模板仓库的 tarball（供离线安装使用）
- 包含所有模板文件和示例项目

**使用场景**：
- 企业内部网络（无法访问 npm registry）
- 特定版本锁定（需要精确复现某个版本的模板）

### 4.3 安装方式对比

| 方式 | 命令 | 适用场景 |
|------|------|---------|
| npm | `npm install -g lupine` | 有网络，需要最新版本 |
| npx | `npx lupine init` | 临时使用，不全局安装 |
| tarball | `lupine init --from-tarball <url>` | 离线环境、版本锁定 |

## 5. CI/CD 流程（v0.5 规划）

### 5.1 自动化发布

```
代码合并到 main
    ↓
运行测试（单元测试 + 集成测试）
    ↓
构建 CLI 包
    ↓
发布到 npm registry
    ↓
创建 GitHub Release + 上传 tarball
```

### 5.2 多仓库支持（v0.5）

- 支持从多个模板仓库初始化项目
- `.lupineconfig.json` 支持 `templateRepo` 字段指定模板源
- 企业内部可搭建私有模板仓库
