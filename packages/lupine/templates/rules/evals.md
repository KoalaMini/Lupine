# 评估门禁标准

## 评估节点

| 节点 | 审查对象 | 审查人 | 通过条件 |
|------|---------|--------|---------|
| Plan 审查 | plans/ | 评估器 | 符合 specs，步骤可执行，粒度合理 |
| Code 审查 | 代码 + 测试 | 评估器 | 符合 plan，测试通过，覆盖边界 |

## 评估维度

1. **需求一致性** — 实现是否与 spec / plan 一致
2. **代码质量** — 可读性、健壮性、性能
3. **测试覆盖** — 核心路径、边界条件、异常场景
4. **规范遵守** — 编码规范、Git 规范
