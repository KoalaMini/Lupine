# 技术栈与编码规范

## 技术栈
- 语言: Markdown
- 版本管理: Git
- 项目管理: GitHub Issues + PR

## 文件约定
- `specs/` — 需求规格说明书（.md）
- `plans/` — 技术设计方案（.md）
- `reviews/` — 审查报告（.md）


## 高质量编码原则

### 单一职责
- 一个文件只做一件事：编排层仅编排，执行单元仅执行
- 模块间通过环境变量 + 返回值通信，不跨模块写全局变量

### 调研先行
- 给方案前先查清现有数据格式与规范（如 SKILL.md 的 YAML front matter）
- 优先扩展已有的数据结构，避免另起文件
- 确认接口契约再动手，不凭空假设
